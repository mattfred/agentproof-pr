# AgentProof PR

### Ships High-Quality PRs in under 5 minutes.

AgentProof PR is a GitHub Action that automatically scores pull requests for human-readiness. It ensures that PRs—especially those generated by AI coding agents—meet your team's quality standards before a human developer ever lays eyes on them.

## 🎯 Who it is for
- **Engineering Teams** looking to automate the "first pass" of PR reviews.
- **AI-Powered Developers** who want to ensure their agent's output is complete and context-aware.
- **Maintainers** of busy open-source projects who need to filter out low-quality contributions.

## 🚀 Why it exists
AI agents are fast, but they often miss "the small things": linking to tickets, adding test evidence, including screenshots for UI changes, or cleaning up debug logs. AgentProof PR catches these before they reach a human, reducing review cycles and cognitive load.

## ⚡ Quick Start (Under 5 Minutes)

Add `.github/workflows/agentproof.yml` to your repo:

```yaml
name: AgentProof PR
on:
  pull_request:
    types: [opened, edited, synchronize]

jobs:
  check:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
      issues: write
    steps:
      - uses: actions/checkout@v4
      - uses: mattfred/agentproof-pr@v0.1.0 # We recommend pinning to tags
        with:
          token: ${{ github.token }}
          comment_on_pr: true
```

## 🛠 Required Permissions
The action requires the following permissions in your GitHub workflow:
- `contents: read`: To check the code for placeholders and UI changes.
- `pull-requests: write`: To post readiness reports as PR comments.
- `issues: write`: Required for certain API interactions related to PRs.

## ⚙️ Configuration Overview
Create a `.agentproof.yml` file in your repository root to customize the rules.

```yaml
minimum_score: 80
fail_on_blockers: true
blocking_rules:
  - linked_ticket
  - summary

weights:
  acceptance_criteria: 30
  test_evidence: 30
```

See [docs/configuration.md](docs/configuration.md) for full options.

## ✅ Supported Checks
- **Linked Ticket**: Ensures the PR references a tracking issue (Jira, GitHub, etc.).
- **Summary & Context**: Validates that the PR description explains *what* and *why*.
- **Acceptance Criteria**: Checks if the PR explicitly addresses the task's requirements.
- **Test Evidence**: Encourages including proof that the changes actually work.
- **UI Screenshots**: Detects UI changes and requires visual proof.
- **Placeholder Detection**: Flags `TODO`, `console.log`, and other temporary markers.
- **CI Status**: Integrates with your existing CI results.

## 📤 Outputs
- **Readiness Score**: A 0-100 score based on your team's weights.
- **PR Comment**: Detailed feedback directly on the pull request.
- **Job Status**: Fails the build if the score is too low or blockers are hit.

## 🔒 Privacy & Security Note
AgentProof PR runs entirely within your GitHub Actions environment. No code, metadata, or PR content is ever sent to external servers.
